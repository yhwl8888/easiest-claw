# 文本编码治理

为避免“中文变乱码”“同一文件反复出现编码差异”等问题，仓库采用以下约束：

## 统一规则

- 文本文件统一使用 `UTF-8`（无 BOM）
- 代码与配置文件统一使用 `LF` 行尾
- 跨平台行尾由 `.gitattributes` 统一处理
- 编辑器默认编码由 `.editorconfig` 统一约束

## 本地检查命令

```bash
# 一次性安装 Git 提交钩子（建议每台机器执行一次）
npm run setup:githooks

# 检查：必须通过
npm run check:encoding

# 自动修复：仅移除 UTF-8 BOM
npm run fix:encoding
```

安装钩子后，每次 `git commit` 会自动执行编码检查。

## CI 保障

- Windows / macOS 构建流程会先执行 `npm run check:encoding`
- 编码检查不通过时，构建会直接失败，阻止问题进入主分支

## Windows 开发建议

- 终端建议使用 UTF-8：
  - PowerShell: `chcp 65001`
- 避免使用会隐式转码的文本处理方式，优先使用 IDE 保存
- 批量改文本前先执行一次 `npm run check:encoding`
